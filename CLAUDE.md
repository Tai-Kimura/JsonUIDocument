# JsonUIDocument Development Guide

## Documentation Reference

When looking for information about JsonUI components, attributes, or features:
- **SwiftJsonUI Wiki**: `~/resource/SwiftyJsonUI_wiki/`
- **KotlinJsonUI Wiki**: `~/resource/KotlinJsonUI_wiki/`

Always check these wiki directories first for accurate documentation.

## Overview

JsonUI Documentation website built with ReactJsonUI (Next.js + Tailwind CSS).

## Project Structure

```
src/
├── Layouts/          # JSON layout files
├── Strings/          # Localization files (en.json, ja.json)
├── Styles/           # Style definition files
├── app/              # Next.js app router pages
├── components/       # React components
├── generated/        # Auto-generated files (DO NOT EDIT)
└── viewmodels/       # ViewModel files
```

## Development Workflow

### Build Commands
```bash
# Build generated files from JSON layouts
./rjui_tools/bin/rjui build

# Watch mode for development
./rjui_tools/bin/rjui watch

# Start Next.js dev server
npm run dev
```

### Configuration
- `rjui.config.json` - ReactJsonUI configuration
- Languages: English (en), Japanese (ja)
- Uses Tailwind CSS

## Important Rules

### Writing JSON Layout Files
- **NEVER guess or assume JSON attributes or component properties**
- If the information is not in the current context, **ALWAYS check the wiki first**:
  - `~/resource/SwiftyJsonUI_wiki/`
  - `~/resource/KotlinJsonUI_wiki/`
- Do not invent attribute names or values based on assumptions

### Creating New Components/Views
- **ALWAYS use CLI commands to create new components or views**
- Never manually create JSON layout files or viewmodels
```bash
# Create a new view
./rjui_tools/bin/rjui g view <view_name>

# Create a new component
./rjui_tools/bin/rjui g component <component_name>
```

### Creating Custom Converters
- If existing components cannot handle the requirement, create a custom converter
```bash
# Create a new converter
./rjui_tools/bin/rjui g converter <converter_name>
```

### Auto-Generated Files
- **NEVER directly edit files in `src/generated/`** - These are auto-generated by `rjui build`
- All changes should be made in:
  - JSON layout files (`src/Layouts/*.json`)
  - Style files (`src/Styles/*.json`)
  - String files (`src/Strings/*.json`)
  - ViewModel files (`src/viewmodels/`)

### rjui_tools
- Location: `rjui_tools/` (project root)
- Main tool: `./rjui_tools/bin/rjui`

### Updating rjui_tools
- **IMPORTANT**: When modifying files under `rjui_tools/`, always apply the same changes to the ReactJsonUI library
- ReactJsonUI library location: `~/resource/ReactJsonUI/rjui_tools/`
- After updating ReactJsonUI, commit and push the changes:
```bash
cd ~/resource/ReactJsonUI
git add -A
git commit -m "Your commit message"
git push origin main
```

## Related Projects

JsonUI is a cross-platform UI framework that generates native views from the same JSON definition:

- **ReactJsonUI**: `~/resource/ReactJsonUI` - React/Next.js version
- **SwiftJsonUI**: `~/resource/SwiftJsonUI` - iOS/Swift version (UIKit & SwiftUI)
- **KotlinJsonUI**: `~/resource/KotlinJsonUI` - Android/Kotlin version (XML & Compose)

All three projects share the same JSON schema, allowing a single JSON layout to render identical UI across web, iOS, and Android platforms.
