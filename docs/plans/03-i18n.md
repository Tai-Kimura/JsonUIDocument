# 03. 多言語対応（日英同時スタート）

> **本計画書の i18n は jui 共通 strings 機構に乗る**点が従来の ReactJsonUI 単独案と異なる。
> 多言語文字列の正本は `docs/screens/layouts/Resources/strings.json`（jui の `strings_file`）であり、
> `jui build` によって各プラットフォーム向けに配布される。詳細は `02-tech-stack.md`。

## 1. 方針

- **初期対応言語**: `en`（デフォルト） / `ja`
- リリース時点で**全ページ日英同時公開**
- Web の URL は言語で分岐させない（クライアント側で StringManager が切替）
- iOS / Android はシステム言語をデフォルトに、アプリ内トグルで切替
- 言語切替は localStorage / UserDefaults / SharedPreferences 保持

## 2. 正本: `docs/screens/layouts/Resources/strings.json`

jui ワークフローの正本。キーごとに全言語を持つ。

```json
{
  "nav_learn":        { "en": "Learn",       "ja": "学ぶ" },
  "nav_guides":       { "en": "Guides",      "ja": "ガイド" },
  "nav_reference":    { "en": "Reference",   "ja": "リファレンス" },
  "nav_platforms":    { "en": "Platforms",   "ja": "プラットフォーム" },
  "nav_tools":        { "en": "Tools",       "ja": "ツール" },
  "nav_concepts":     { "en": "Concepts",    "ja": "概念" },
  "hero_title":       { "en": "JsonUI",      "ja": "JsonUI" },
  "hero_subtitle":    { "en": "Cross-platform UI in JSON", "ja": "JSON で書くクロスプラットフォーム UI" },
  "ref_label_title":  { "en": "Label",       "ja": "Label" }
}
```

**Layout JSON 内での参照構文** (`@string/key_name`):

```json
{ "type": "Label", "text": "@string/hero_title", "style": "heading_1" }
```

`@string/hero_title` は `jui build` 時に各プラットフォームの StringManager 参照形式に変換される:

| プラットフォーム | 変換後 |
|---|---|
| Web (rjui) | `{StringManager.currentLanguage.heroTitle}` |
| iOS (sjui) | `SJUIViewCreator.stringFromKey("hero_title")` |
| Android (kjui) | `@string/hero_title`（XML）or `stringResource(R.string.hero_title)`（Compose） |

## 3. `jui build` による配布

`jui build` は `docs/screens/layouts/Resources/strings.json` を各プラットフォームに応じた形式で配布:

| プラットフォーム | 出力先 | 形式 |
|---|---|---|
| Web | `jsonui-doc-web/src/Strings/en.json` / `ja.json` | 言語別 flat JSON |
| iOS | `jsonui-doc-ios/JsonUIDoc/Resources/Localizable.strings`（言語別） | `.strings` フォーマット |
| Android | `jsonui-doc-android/app/src/main/res/values/strings.xml` / `values-ja/strings.xml` | Android XML |

変換スクリプトは `jui_tools` 側に実装（未実装であれば Phase 1 で追加コントリビュート）。

### 3.1 Web 側の生成物（例）

```json
// jsonui-doc-web/src/Strings/en.json （jui build が生成、.gitignore）
{
  "nav_learn": "Learn",
  "nav_guides": "Guides",
  "hero_title": "JsonUI",
  "ref_label_title": "Label"
}
```

rjui の `StringManager` がこのファイルを読み込み、現在言語のキーを解決する（新規に実装）。

## 4. 文字列キー命名規則

snake_case、プレフィックスで分類:

| プレフィックス | 対象 |
|----|----|

| `hero_*` | トップ Hero |

## 5. 本文コンテンツの扱い

ドキュメントサイトの特殊事情: 段落レベルの長文が多数発生する。Strings に詰めると肥大化する。

### 5.1 短い UI 文言 → Strings

ボタン、見出し（H1〜H4 相当）、短い説明文、エラーメッセージ。`Resources/strings.json` に直接書く。

### 5.2 長文（記事本文） → 別データ形式

長文の記事本文（Learn / Guides の本文など）は、**`dataFlow.repositories` 経由で取得する構造**を採る:

```jsonc
// docs/screens/json/learn/what-is-jsonui.spec.json
{
  "dataFlow": {
    "repositories": [
      {
        "name": "DocContentRepository",
        "methods": [
          {
            "name": "fetchArticle",
            "params": [
              { "name": "id",   "type": "String" },
              { "name": "lang", "type": "String" }
            ],
            "returnType": "DocArticle",
            "isAsync": true
          }
        ]
      }
    ]
  }
}
```

記事本文データの置き場所（全プラットフォーム共通）:

```
docs/content/
├── en/
│   ├── learn/
│   │   ├── what-is-jsonui.json
│   │   └── installation.json
│   ├── guides/
│   └── concepts/
└── ja/
    ├── learn/
    ├── guides/
    └── concepts/
```

データフォーマット例（`docs/content/en/learn/what-is-jsonui.json`）:

```json
{
  "id": "learn/what-is-jsonui",
  "title": "What is JsonUI?",
  "sections": [
    {
      "type": "heading",
      "level": 2,
      "text": "Why JSON?"
    },
    {
      "type": "paragraph",
      "text": "JsonUI defines UI in JSON so that iOS, Android, and Web can share a single source of truth..."
    },
    {
      "type": "code",
      "language": "json",
      "code": "{ \"type\": \"Label\", \"text\": \"Hello\" }"
    }
  ]
}
```

### 5.3 Layout JSON 側の描画

`View style="prose"` + `Collection` で sections を展開（`prose` スタイルは `docs/screens/styles/prose.json` で Tailwind `prose` クラスを割り当てる）。`Prose` カスタム型は作らない（`02-tech-stack.md §6.0`）。

```json
{
  "type": "View",
  "style": "prose",
  "child": [
    {
      "type": "Collection",
      "items": "@{article.sections}",
      "cellIdProperty": "id",
      "cellClasses": ["cells/prose_section"]
    }
  ]
}
```

`cells/prose_section.json` 内部で `section.type` により heading / paragraph / code を分岐（`displayLogic` で可）。

### 5.4 本文データのデプロイ

- Web: `docs/content/` を `jsonui-doc-web/public/content/` にシンボリックリンクまたは `jui build` カスタムフックでコピー、fetch で取得
- iOS: `docs/content/` を Xcode project の Resources として bundle に含める
- Android: `docs/content/` を `app/src/main/assets/content/` に配置

上記のコピーは `jui.config.json` の追加 `additionalResources`（将来提案）か、Phase 2 の自作スクリプトで対応。

## 6. Strings validation

`jsonui-doc-web/scripts/validate-strings.ts`:

```ts
import fs from "fs/promises";

async function main() {
  const src = JSON.parse(
    await fs.readFile("../docs/screens/layouts/Resources/strings.json", "utf8")
  );

  const langs = new Set<string>();
  for (const val of Object.values<any>(src)) {
    for (const lang of Object.keys(val)) langs.add(lang);
  }

  const errors: string[] = [];
  for (const [key, val] of Object.entries<any>(src)) {
    for (const lang of langs) {
      if (!(lang in val) || val[lang] === "") {
        errors.push(`Key '${key}' missing for lang '${lang}'`);
      }
    }
  }

  if (errors.length) {
    console.error(errors.join("\n"));
    process.exit(1);
  }
}

main();
```

`package.json` scripts:

```json
{
  "scripts": {
    "validate:strings": "tsx scripts/validate-strings.ts",
    "prebuild": "npm run validate:strings && ..."
  }
}
```

CI で必須化。

## 7. 翻訳ワークフロー

1. 開発者が**英語を先に書く**（権威テキスト）
2. `Resources/strings.json` にキー追加（両言語値を同時に）
3. 本文コンテンツは `docs/content/en/` に作成 → `docs/content/ja/` に翻訳
4. `jui verify` と `validate:strings` を通す
5. `jui build` で配布

## 8. 言語切替 UI

```json
// docs/screens/layouts/common/language_toggle.json
{
  "type": "View",
  "orientation": "horizontal",
  "child": [
    {
      "type": "Label",
      "text": "EN",
      "fontColor": "@{currentLanguage == 'en' ? 'primary' : 'gray'}",
      "onClick": "onSetEnglish"
    },
    { "type": "Label", "text": "|" },
    {
      "type": "Label",
      "text": "JA",
      "fontColor": "@{currentLanguage == 'ja' ? 'primary' : 'gray'}",
      "onClick": "onSetJapanese"
    }
  ]
}
```

ViewModel 側（`jui g project` 生成後に実装）:

```ts
onSetEnglish() {
  StringManager.setLanguage("en");
  this.currentLanguage = "en";
  localStorage.setItem("jsonui-language", "en");
}
```

Web は `StringManager`、iOS / Android もそれぞれのネイティブ StringManager 相当に委譲。

## 9. SEO とアクセシビリティ（Web）

- `<html lang={lang}>` を動的に更新
- 主要ページの `<meta name="description">` は言語切替時に書き換え
- `<link rel="alternate" hreflang="ja">` は URL 分岐を採用した v2 で導入

## 10. 日付・数値のローカライズ

リリースノート日付などは `Intl.DateTimeFormat` で整形（Web）。iOS は `DateFormatter`、Android は `DateTimeFormatter`。ViewModel で言語依存の整形を行い、Layout には整形済み文字列だけ渡す。

## 11. 翻訳の網羅対象（初期リリース）

- すべての UI 文言（`Resources/strings.json`）
- すべての `docs/content/` の記事
- Reference の「属性 description」は MCP / attribute_definitions.json の英語をベースに、`docs/data/attribute-overrides/*.json`（`14` 参照）で日本語追記
- Reference の長めのディスカッション部分は英語先行でリリース可（ただし本文 80% 以上が日本語同梱）

## 12. 将来の追加言語

`jui.config.json` に言語リストを保持する提案（現状の `jui_tools` では strings ファイル内の言語キーから自動検出）。
追加時は `Resources/strings.json` の値オブジェクトにキーを追加 + `docs/content/{lang}/` を用意するだけ。
